# Skill: Protocolo de Apresentação e Deploy Seguro

Regras rigorosas para apresentação de protótipos em reuniões comerciais.
*   **Desenvolvimento:** Construir o front-end em localhost para testes de animação e lógica [14].
*   **Staging:** Fazer o deploy do código para uma URL oculta (Vercel ou Netlify) dias antes da apresentação [14].
*   **Regra de Ouro da Execução:** Durante o compartilhamento de tela no Meet, utilizar **exclusivamente o link de produção (Vercel)**. É terminantemente proibido apresentar rodando em ambiente local para evitar quebras de frame rate e travamento do Node.js ao vivo [14].
